Activity 1.3.4.

We now will explore mathematical variables.

(a)

Unlike CS variables, mathematical variables represent an unknown value or arbitrary value, rather than storing a specific piece of data. These variables are often represented by single letters rather than the long semantic names used for CS variables. For example, SageMath always assumes x is a mathematical variable named \(x\text{.}\)

Other math variables are declared using by using the var() tool. For example, a = var("a") declares a to be the variable named \(a\text{,}\) and b,c = var("b c") declares both b and c to be variables.

Use a Code cell to declare \(y\) and \(z\) to be variables.

(b)

Math variables may be manipulated just like literal values or values store in CS variables. But since they don't store specific values, these combinations of variables represent an expression.

Use the show() tool to display the expression \(x^2+4yz\) using a Code cell. Remember that \(4yz=4\times y\times z\) requires the * operator to perform multiplication.

(c)

An expression can itself be stored in a CS variable using the assignment operator =; for example, quadratic = x^2-4*x+4 saves the expression \(x^2-4x+4\) using the name quadratic.

Assign the expression \(x^2+4yz\) to a CS variable with the name my_expression.

(d)

Math variables and expressions are useful for describing how data should be manipulated before the values of those data are known.

For example, suppose we now want to assume \(x=6,y=0,z=-3\) and compute the corresponding value for the expression \(x^2+4yz\) saved as my_expression. You can check that \(6^2+4(0)(-3)=36\text{,}\) either by hand or evaluating 6^2+4*0*(-3). But to have SageMath compute this for you immediately, you could use the following syntax: my_expression(x=6,y=0,z=-3).

Use a Code cell to substitute the values \(x=1,y=2,z=3\) into the expression \(x^2+4yz\text{,}\) and confirm that the resulting value is equal to \(1^2+4(2)(3)\text{.}\)
